# Telegram-бот для управления рестораном

Бот для автоматизации работы персонала ресторана: графики смен, мотивация (премии/штрафы), учёт чаевых, управление ролями сотрудников. Интеграция с Google Sheets для синхронизации данных.

## Возможности

- **Ролевая модель** – до 3 ролей на сотрудника (официант, бартендер, менеджер, кальянный мастер и др.)
- **Графики смен** – назначение смен по шаблонам (например, 3/3, 5/2), добавление/удаление отдельных смен, синхронизация с Google Sheets
- **Мотивация** – начисление премий, штрафов, жёлтых карт с записью в таблицу
- **Чаевые** – автоматический расчёт по коэффициентам (фиксированные 5% для менеджеров, долевое распределение для официантов/бартендеров/помощников)
- **Управление сотрудниками** – регистрация через бота, подтверждение роли менеджером
- **Аналитика** – личный кабинет сотрудника (мои смены, мои чаевые, моя мотивация)
- **Автоматические бэкапы** SQLite базы данных

## Технологии

- Python 3.9+
- aiogram 3.x (асинхронный фреймворк)
- aiosqlite (асинхронная работа с SQLite)
- gspread + google-auth (Google Sheets API)
- python-dotenv (конфигурация)

## Установка и запуск

### 1. Клонирование репозитория

```bash
git clone https://github.com/your-repo/restaurant-bot.git
cd restaurant-bot
```

### 2. Установка зависимостей

```bash
python -m pip install -r requirements.txt
```

### 3. Настройка окружения

Создайте файл `.env` (на основе `.env.example`):

```env
BOT_TOKEN=ваш_токен_бота
OWNER_ID=123456789
SCHEDULE_SPREADSHEET_ID=1qXt0kKVhTQpi3MJaVCinZG8a0pxZISxUuLNvU-kXkcY
MOTIVATION_SPREADSHEET_ID=1UAeDQksHXDMmoUwXYjRyVe5Ic1Pza48cHT2u3LEuEnA
TIPS_SPREADSHEET_ID=ваш_id_таблицы_для_чаевых
```

- `BOT_TOKEN` – получить у [@BotFather](https://t.me/BotFather)
- `OWNER_ID` – ваш Telegram ID (можно узнать командой `/myid` у бота после запуска)
- ID таблиц Google Sheets (необходимо создать копии [шаблонов](#google-sheets-шаблоны))

### 4. Настройка Google Sheets API

1. Перейдите в [Google Cloud Console](https://console.cloud.google.com/), создайте проект.
2. Включите **Google Sheets API** и **Google Drive API**.
3. Создайте **сервисный аккаунт**, скачайте `credentials.json` в формате JSON.
4. Положите файл `credentials.json` в корневую папку проекта.
5. Предоставьте сервисному аккаунту доступ к вашим таблицам (скопируйте email из `client_email` и добавьте в права на редактирование).

### 5. Подготовка таблиц Google Sheets

Создайте таблицы со следующими листами:

- **График смен** – лист `ШТАТ` (шапка: отделы), листы месяцев (например `05 МАЙ 2026`) с датами.
- **Мотивация** – лист с датами в первой строке, сотрудниками в первом столбце.
- **Чаевые** – таблица создаётся автоматически ботом.

Готовые шаблоны можно получить по запросу в поддержку.

### 6. Инициализация базы данных

При первом запуске бот сам создаст файл `bot_database.db` и необходимые таблицы. Если у вас есть старые JSON-файлы (`users.json`, `motivation.json` и т.д.), запустите миграцию:

```bash
python migrate_to_sqlite.py
```

### 7. Запуск бота

```bash
python main.py
```

Для production рекомендуется использовать `systemd` или `supervisor`.

## Команды бота

| Команда | Кто может | Описание |
|---------|-----------|----------|
| `/start` | Все | Регистрация / вход |
| `/cancel` | Все | Отмена текущего действия |
| `/myid` | Все | Узнать свой Telegram ID |
| `/setadmin` | Только owner (в коде) | Назначить себя владельцем |
| `/schedules` | Менеджер+ | Список типов графиков |
| `/addschedule` | Менеджер+ | Добавить новый тип графика |
| `/newshift` | Менеджер+ | Поставить смены на месяц по шаблону |
| `/addshift` | Менеджер+ | Добавить одну смену |
| `/delshift` | Менеджер+ | Удалить смену |
| `/myshift` | Сотрудник | Мои смены |

Также доступны кнопки в меню.

## Роли и права

Роли определены в `constants.py`:

| Роль | Доступ |
|------|--------|
| 👑 Owner | Всё (управление ролями, настройки, аналитика) |
| 🎖 Сказочный полковник | Всё, кроме управления ролями |
| 📋 Менеджер | Графики, мотивация, добавление сотрудников |
| ⭐ Старший смены | Ввод чаевых |
| 🍽 Официант / 🍹 Бартендер / 🤝 Встреч. менеджер / 🙋 Помощник / 💨 Кальянный мастер | Свой график, свои чаевые, мотивация |

У сотрудника может быть до 3 ролей. Основная роль – первая в списке – определяет меню.

## Структура базы данных (SQLite)

- `users` – сотрудники (`user_id`, `fullname`, `role`, `roles` (JSON), `username`)
- `motivation` – записи мотивации (премии, штрафы)
- `tips` – чаевые по дням (`date`, `total`, `breakdown` (JSON))
- `schedules` – типы графиков (`name`, `work`, `rest`)
- `settings` – настройки (`key`, `value`) – активные листы Google Sheets

## Автоматические бэкапы

- Бот создаёт бэкапы базы данных каждые 24 часа в папку `backups/`.
- Имена файлов: `bot_database_ГГГГ-ММ-ДД_ЧЧ-ММ-СС.db`.
- Старые бэкапы (старше 7 дней) удаляются автоматически.
- Можно запустить вручную: `python backup_db.py`.

## Устранение неполадок

### Бот не запускается: `ModuleNotFoundError: No module named 'middleware'`

Создайте папку `middleware` и файлы `__init__.py`, `role_check.py` (см. код в репозитории).

### Ошибка доступа к Google Sheets

Проверьте:
- `credentials.json` лежит в корне и имеет правильный формат.
- Сервисному аккаунту предоставлен доступ к таблице (скопируйте email из файла и добавьте в редакторы).

### Ошибка при записи чаевых: сотрудник не найден в таблице

Убедитесь, что в активном листе графика в первом столбце есть точное ФИО сотрудника (как в `users.json`).
